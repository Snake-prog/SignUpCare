### Написание программного кода

Разработка программного кода сервиса выполнялась с использованием классического стека серверного веб-приложения на базе Django с серверной генерацией HTML через Django Template Language и клиентской интерактивностью на JavaScript и jQuery. Архитектура решения ориентирована на надёжность, переносимость и простоту развёртывания, что обусловило выбор SQLite в качестве базы данных и обычных Django View.

---

#### Архитектурный подход к разработке

Программный код разрабатывался с применением принципов:

* разделения ответственности между слоями системы;
* модульности приложений Django;
* повторного использования базовых абстракций;
* изоляции бизнес-логики от представлений.

Проект разделён на отдельные Django-приложения (например, `user`, `care`), что обеспечивает независимое развитие функциональных модулей.

---

#### Организация структуры проекта

В проекте реализована многоуровневая структура:

* **apps/** - бизнес-домены системы
* **abstractions/** - базовые классы моделей, админ-панелей и представлений
* **configuration/** - настройки Django, маршрутизация, WSGI/ASGI
* **templates/** - HTML-шаблоны пользовательского интерфейса
* **static/** - статические ресурсы
* **utils/** - вспомогательные инструменты

Такой подход упрощает сопровождение проекта и расширение функциональности.

---

#### Реализация серверной логики

Серверная часть построена на обычных Django View (Class-Based Views). Это позволило:

* упростить архитектуру приложения;
* уменьшить накладные расходы на сериализацию;
* обеспечить прямую связку backend-логики и HTML-шаблонов.

Бизнес-логика вынесена в отдельные слои:

* facades - сценарии бизнес-операций;
* interactors - обработка действий пользователя;
* selectors - выборки данных.

---

#### Реализация работы с базой данных

Для взаимодействия с базой данных используется Django ORM.
Реализованы:

* модели предметной области;
* миграции структуры базы данных;
* валидация данных на уровне моделей.

SQLite выбран как переносимое решение, не требующее отдельного сервера БД.

---

#### Реализация пользовательского интерфейса

Интерфейс реализован через серверные шаблоны Django с использованием Bootstrap для адаптивности.

Клиентская логика включает:

* обработку событий через jQuery;
* сохранение пользовательских настроек в LocalStorage;
* озвучивание интерфейса через SpeechSynthesis API для пользователей с ограниченными возможностями.

---

#### Реализация административной панели

Для административного интерфейса используется Jazzmin, что позволило:

* улучшить внешний вид стандартного Django Admin;
* повысить удобство работы администраторов;
* ускорить разработку инструментов управления данными.

---

#### Управление конфигурацией

Для разделения настроек окружений используется Django-configurations, что обеспечивает:

* разделение development и production конфигураций;
* безопасное хранение настроек;
* удобное масштабирование проекта.

---

#### Качество и сопровождаемость кода

При написании кода соблюдались требования:

* единый стиль кодирования;
* читаемость и прозрачность логики;
* минимизация дублирования;
* подготовка к масштабированию и расширению.
